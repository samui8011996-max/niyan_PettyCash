# 🔄 本次升級說明 (v3.0 · 雙 Sheet 架構)

## 🆕 這次新增的功能

### 1. 試算表拆成兩個分頁

**📋 零用金收支(主表)** — 繼續保持清爽
- 便當款每天只記 1 筆總額:`當日便當款 NT$720`
- 補貼款每人記 1 筆:`淑莉-午餐補助 NT$100`(以後對帳知道補給誰)

**🍱 今日便當(新分頁)** — 所有人的訂餐明細,永久保留
- 每人每天 1 筆:誰、吃什麼、少飯與否、金額、類型、登記人
- 未來可以做流水帳統計、看誰最常吃雞排等等

### 2. 網頁看板升級

**點餐頁:**
- 即時看板顯示誰訂了什麼 (✓ 已訂 / 尚未選擇)
- 三大金額一目了然:便當款 / 補貼款 / 總餐費

**記錄頁:**
- 「🍱 便當結算查詢」可以切任何日期,看當天結算

## ⚙️ 你需要做的更新

### 步驟 1:更新 index.html

1. 進你的 GitHub repo
2. 刪除舊的 `index.html`
3. 上傳新的 `index.html`(直接拖到根目錄,**不要放進資料夾**)
4. 等 1 分鐘讓 GitHub Pages 重新建置

### 步驟 2:更新 Apps Script 程式碼

⚠️ **重要:務必用「編輯現有部署」,不要「新增部署作業」,否則網址會變!**

1. 打開你的試算表 → 擴充功能 → Apps Script
2. 把舊程式碼**全部刪掉**,貼上新的 `apps-script-code.js` 全部內容
3. 按 Ctrl+S 存檔
4. 右上角 **部署 → 管理部署作業**
5. 點那個現有部署右上的 **鉛筆圖示 ✏**
6. **版本** 下拉選 **新版本**
7. 按 **部署**
8. 關閉視窗(網址不變,不用重抄)

### 步驟 3:試算表分頁處理

**好消息:什麼都不用做!**

Apps Script 會自動偵測——第一次上傳訂餐時,如果「今日便當」分頁還不存在,會**自動建立**,標題列也會自動設好。

如果你想先手動建好看看長什麼樣也可以,但不是必要的。

### 步驟 4:測試

1. 打開你的網頁
2. 在點餐頁選 2-3 個人測試(記得混一個選「午餐補助」)
3. 按上傳
4. **去試算表看看**:
   - 「零用金收支」分頁:應該看到 1 筆「當日便當款」+ 補助的人各自 1 筆「XX-午餐補助」
   - 「今日便當」分頁(會自動出現!):看到所有人的明細

## ❓ 可能會遇到的問題

**Q: 之前測試產生的舊格式記錄怎麼辦?**
A: 直接到試算表手動刪掉就好。新格式會和舊格式共存不會出錯,但看了心煩所以建議清掉。

**Q: 「今日便當」分頁名字一定要這樣嗎?**
A: 是的,要跟 Apps Script 裡的 `SHEET_LUNCH = '今日便當'` 完全一致。如果想改,兩邊一起改。

**Q: 舊版本的便當記錄(寫在主表的 `妍儒【便當-雞排】`)會不會出現在新看板?**
A: 不會,新看板只讀「今日便當」分頁。建議把舊記錄從主表刪掉。

**Q: 沒有選「午餐補助」的人(空白)不用處理嗎?**
A: 對,空白的人什麼都不寫,雙表都跳過。
