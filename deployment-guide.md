# 🚀 零用金系統上架教學(公司全員共用版)

跟著做完,你會得到一個像 `https://arte.github.io/petty-cash` 的固定網址,公司每個人手機、電腦都能打開用,選餐即時同步。

**預計時間:10~15 分鐘(第一次會慢一點,很正常)**

---

## 📋 準備清單

做之前先確認你有這三樣:

- [ ] **Google 帳號**(你有試算表就一定有)
- [ ] **GitHub 帳號**(沒有?看下方「附錄 A」5 分鐘申請)
- [ ] **你的試算表連結**(已經有了)

---

## 步驟一:準備試算表欄位 ⏱ 1 分鐘

打開你的零用金試算表,確認第 1 列(標題列)是這樣:

| A | B | C | D | E | F | G | H |
|---|---|---|---|---|---|---|---|
| 日期 | 項目 | 類型 | 收入金額 | 支出金額 | 經手人 | 備註 | 總餘額 |

**在 H2 儲存格輸入公式**(自動計算餘額):
```
=IFERROR(SUMIF($D$2:D2,">0",$D$2:D2)-SUMIF($E$2:E2,">0",$E$2:E2),0)
```
往下拖曳到 H100 以上,之後新增的列會自動有公式。

---

## 步驟二:部署 Google Apps Script ⏱ 5 分鐘

### 2-1. 開啟 Apps Script 編輯器

在你的試算表中,點上方選單 **擴充功能 → Apps Script**。會跳出新分頁。

### 2-2. 貼上程式碼

把預設那幾行程式碼**全部刪掉**,然後打開我給你的 `apps-script-code.js` 檔案,複製全部內容貼進去。

按 **Ctrl+S**(或 ⌘+S)存檔,專案名稱可以取「零用金 API」。

### 2-3. 部署

1. 點右上角藍色的 **部署** 按鈕 → **新增部署作業**
2. 左側齒輪 ⚙ → 選 **網頁應用程式**
3. 填寫:
   - **說明**:零用金 v2(隨便打)
   - **執行身分**:我 (你的 Gmail)
   - **誰可以存取**:**任何人** ⚠️(這步超重要!)
4. 按 **部署**

### 2-4. 授權

第一次會跳「需要授權」。按順序做:

1. **授予存取權** → 選你的 Google 帳號
2. 會出現「Google 尚未驗證這個應用程式」警告 → **別怕**,這是你自己寫的程式
3. 點左下角 **進階** → **前往「零用金 API」(不安全)**
4. 最下面 → **允許**

### 2-5. 複製網址 ⭐

部署成功會跳出一個框框,有個網址長這樣:

```
https://script.google.com/macros/s/AKfycbxxxxxxxxxxxxxxxxx/exec
```

**按旁邊的 📋 複製**,先貼到記事本裡別弄丟!

> ⚠️ **如果以後改了程式碼**:要回到 **部署 → 管理部署作業 → 鉛筆圖示 → 版本選「新版本」→ 部署**,網址不會變,但程式碼會更新。如果直接「新增部署作業」會拿到新網址,之前設定的人就斷線了。

---

## 步驟三:上傳網頁到 GitHub Pages ⏱ 5 分鐘

### 3-1. 建立 Repository(儲存庫)

1. 登入 GitHub,右上角 **+** → **New repository**
2. 填寫:
   - **Repository name**:`petty-cash`(或任何你喜歡的名字)
   - **Public**:✅ 選這個(GitHub Pages 免費版需要公開,不用擔心,沒人會知道你的網址)
   - ✅ 勾選 **Add a README file**
3. 點綠色的 **Create repository**

### 3-2. 上傳 index.html

1. 在你剛建立的 repo 頁面,點 **Add file → Upload files**
2. 把我給你的 `index.html` 檔案拖進去
3. 下方 **Commit changes** 按綠色按鈕

### 3-3. 啟用 GitHub Pages

1. 在 repo 頁面上方,點 **Settings**(齒輪圖示)
2. 左側選單 → **Pages**
3. **Source** 設定:
   - **Branch**:選 `main`
   - 資料夾:`/ (root)`
   - 按 **Save**
4. 等約 1 分鐘,頁面上方會出現:
   > Your site is live at `https://你的帳號.github.io/petty-cash/`

**把這個網址抄下來**,這就是你公司全員要用的網址了!

---

## 步驟四:首次設定 ⏱ 1 分鐘

1. 打開上面那個 GitHub Pages 網址
2. 點右上角齒輪圖示 ⚙
3. **Web App URL** 欄位:貼入步驟 2-5 複製的那個 `https://script.google.com/...` 網址
4. **試算表連結** 欄位:貼你的 Google Sheet 網址(選填,方便點擊跳轉)
5. 按 **測試連線**——應該會跳綠色提示「連線成功」
6. 按 **儲存並重新載入**

**完成!** 🎉 現在點餐頁右上角會看到綠色脈動點表示「即時同步中」。

---

## 步驟五:分享給同事 ⏱ 1 分鐘

把 GitHub Pages 網址分給大家,每個人第一次打開:

1. 點右上角 ⚙
2. **只要填 Web App URL**(同一個網址,公司共用)
3. 儲存

之後就都連在一起了。

## 📱 手機使用小技巧

**加到主畫面,像 App 一樣用:**

- **iPhone(Safari)**:打開網址 → 下方分享圖示 → 加到主畫面
- **Android(Chrome)**:打開網址 → 右上 ⋮ → 加到主畫面

加完會有一個圖示在桌面,點開直接用。

## 🎯 即時同步怎麼運作

**場景舉例:**

早上 10:00,妍儒打開網頁,點了「雞排」→ 按上傳 → 試算表出現一筆記錄。

早上 10:05,佳宜打開網頁 → **自動就看到妍儒已經選了雞排**(綠色邊框 + ✓ 已上傳標記),佳宜就知道不用幫她點了。

佳宜選了「鯖魚排」→ 按上傳。

早上 10:10,維妮打開網頁 → 看到妍儒(雞排)和佳宜(鯖魚排)都已經上傳,她只要選自己的。

**20 秒自動同步一次**,也可以按右上角 ↻ 手動同步。

## ❓ 常見問題

<details>
<summary><b>Q: 我沒有 GitHub 帳號怎麼辦?</b></summary>

看下方附錄 A,5 分鐘就能申請好。
</details>

<details>
<summary><b>Q: 同事需要 GitHub 或 Google 帳號嗎?</b></summary>

**都不用**。他們只要打開網址、填一次 Web App URL(你提供給他們)就好。
</details>

<details>
<summary><b>Q: 測試連線失敗?</b></summary>

90% 是這兩個原因:
1. **部署時「誰可以存取」沒設成「任何人」** → 回 Apps Script → 部署 → 管理部署作業 → 編輯 → 改成「任何人」→ 部署
2. **沒授權完成** → 重新跑一次授權流程(步驟 2-4)

如果還是不行,按 F12 打開瀏覽器開發者工具看 Console 錯誤訊息。
</details>

<details>
<summary><b>Q: 有人同時選同一人的便當會衝突嗎?</b></summary>

會「後到的蓋前面的」。但實務上很少發生,因為上傳前大家都會看到目前狀態。
</details>

<details>
<summary><b>Q: 我想修改商品、便當菜單?</b></summary>

開 `index.html`,找到 `const BENTO_MENU =` 或 `const PRODUCTS =`,修改後重新上傳到 GitHub(Add file → Upload → 覆蓋)。約 1 分鐘後網頁會更新。
</details>

<details>
<summary><b>Q: GitHub Pages 真的免費?會不會突然收費?</b></summary>

GitHub Pages 公開 repo 永久免費,從 2008 年用到現在,Microsoft 收購後政策也沒改。算是最穩的免費靜態網頁服務了。
</details>

<details>
<summary><b>Q: 我的試算表和程式碼會被別人看到嗎?</b></summary>

- **試算表**:不會,除非你主動分享
- **index.html**:會(因為公開 repo),但裡面沒有任何敏感資料,全世界都能抄也沒差
- **Apps Script 網址**:等於是你家後門鑰匙,別公開貼網路。但即使外人拿到也只能寫你的試算表(頂多被灌垃圾資料,可以回頭刪掉)
</details>

<details>
<summary><b>Q: 想關閉公開存取怎麼辦?</b></summary>

最簡單:在 Apps Script 點「管理部署作業」→ 封存。網頁就會斷線,但不影響試算表資料。
</details>

## 附錄 A:申請 GitHub 帳號(5 分鐘)

1. 打開 [https://github.com/signup](https://github.com/signup)
2. 填 Email → 密碼 → 使用者名稱(這會出現在網址裡,例如 `arte` 就會變成 `arte.github.io`)
3. 驗證人機驗證
4. Email 收到驗證碼,填入
5. 方案選 **Free**
6. 問卷可以隨便填或跳過

完成!可以回到步驟三繼續。

---

## 附錄 B:Apps Script 程式碼更新方式

以後如果我幫你改了 Apps Script(例如新增功能):

1. 進 Apps Script 編輯器(從試算表 → 擴充功能 → Apps Script)
2. 全選原本程式碼 → 刪除 → 貼上新版
3. 存檔(Ctrl+S)
4. **部署 → 管理部署作業** → 右上角鉛筆圖示 ✏
5. **版本** 下拉選 **新版本**
6. 按 **部署**

⚠️ **重點**:**一定要選「編輯現有部署」不要「新增部署作業」**,不然網址會變!
